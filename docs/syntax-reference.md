# 模板变量与条件表达式 — 完整语法参考

> 本文档涵盖系统支持的所有模板变量语法、条件表达式语法和 `<if>` 标签语法。

---

## 一、变量定义标签

### 1. 随机数变量 `<random>`

```html
<!-- 生成随机数并直接替换标签 -->
<random min="1" max="100" />

<!-- 生成随机数并存储为变量（标签被移除，值可通过 $random:id 引用） -->
<random id="dice" min="1" max="6" />
```

**参数：**
| 参数 | 必选 | 说明 |
|------|:----:|------|
| `min` | ✅ | 最小值（整数） |
| `max` | ✅ | 最大值（整数） |
| `id` | ❌ | 变量名，指定后标签被移除，值存入变量 |

**引用语法：** `$random:变量名`

---

### 2. 计算变量 `<calc>`

```html
<!-- 定义计算变量（标签被移除，值可通过 $calc:id 引用） -->
<calc id="total" expr="cell:角色属性表/攻击力/数值 + $random:dice * 2" />
```

**参数：**
| 参数 | 必选 | 说明 |
|------|:----:|------|
| `id` | ✅ | 变量名 |
| `expr` | ✅ | 计算表达式，支持四则运算和括号 |

**表达式中可引用的值：**
- 纯数字：`42`、`3.14`
- 单元格值：`cell:表名/行名/列名`
- 随机数变量：`$random:变量名`
- 计算变量：`$calc:变量名`
- 最大值变量：`$max:变量名`
- 最小值变量：`$min:变量名`

**引用语法：** `$calc:变量名`

---

### 3. 最大值变量 `<max>`

```html
<!-- 取多个值中的最大值 -->
<max id="highest" values="cell:角色属性表/攻击力/数值, $random:dice, 50" />
```

**参数：**
| 参数 | 必选 | 说明 |
|------|:----:|------|
| `id` | ✅ | 变量名 |
| `values` | ✅ | 逗号分隔的值列表，每个值支持与 `<calc>` 相同的引用语法 |

**引用语法：** `$max:变量名`

---

### 4. 最小值变量 `<min>`

```html
<!-- 取多个值中的最小值 -->
<min id="lowest" values="cell:角色属性表/防御力/数值, $random:dice, 10" />
```

**参数：** 与 `<max>` 相同。

**引用语法：** `$min:变量名`

---

## 二、值替换模板变量

### 1. ORM 查询 `{[db.表名.方法链]}`

```
你有 {[db.背包物品表.where('物品名称', '铁剑').get('数量')]} 把铁剑
```

**支持的方法链：**

| 方法 | 说明 | 返回值 |
|------|------|--------|
| `.where('列名', '值')` | 等值条件 | `TableQueryBuilder`（链式） |
| `.where('列名', '>', 数值)` | 比较条件（支持 `>`/`>=`/`<`/`<=`/`!=`） | `TableQueryBuilder`（链式） |
| `.where('列名', null)` | IS NULL 条件 | `TableQueryBuilder`（链式） |
| `.where('列名', '!=', null)` | IS NOT NULL 条件 | `TableQueryBuilder`（链式） |
| `.orWhere('列名', '值')` | OR 条件（开始新的 OR 分支） | `TableQueryBuilder`（链式） |
| `.orWhere('列名', '>', 数值)` | OR 比较条件 | `TableQueryBuilder`（链式） |
| `.whereIn('列名', [值1, 值2])` | IN 查询 | `TableQueryBuilder`（链式） |
| `.whereNotIn('列名', [值1, 值2])` | NOT IN 查询 | `TableQueryBuilder`（链式） |
| `.whereBetween('列名', 最小值, 最大值)` | BETWEEN 范围查询 | `TableQueryBuilder`（链式） |
| `.whereNull('列名')` | IS NULL 条件 | `TableQueryBuilder`（链式） |
| `.whereNotNull('列名')` | IS NOT NULL 条件 | `TableQueryBuilder`（链式） |
| `.whereLike('列名', '模式')` | LIKE 模糊匹配（`%` 和 `_` 通配符） | `TableQueryBuilder`（链式） |
| `.orderBy('列名', 'ASC')` | 排序（`ASC`/`DESC`） | `TableQueryBuilder`（链式） |
| `.groupBy('列名')` | 分组 | `TableQueryBuilder`（链式） |
| `.having('表达式')` | 分组后过滤（配合 `groupBy` 使用） | `TableQueryBuilder`（链式） |
| `.distinct()` | 去重 | `TableQueryBuilder`（链式） |
| `.limit(数量)` | 限制返回行数 | `TableQueryBuilder`（链式） |
| `.offset(数量)` | 偏移量（配合 `limit` 用于分页） | `TableQueryBuilder`（链式） |
| `.get('列名')` | 获取单个值（第一行指定列） | `string \| number \| null` |
| `.first()` | 获取单行（所有列） | `Record<string, any> \| null` |
| `.list('列名')` | 获取某列的值列表 | `(string \| number)[]` |
| `.all()` | 获取所有行 | `Record<string, any>[]` |
| `.count()` | 计数 | `number` |
| `.sum('列名')` | 求和 | `number` |
| `.avg('列名')` | 求平均值 | `number` |
| `.max('列名')` | 求最大值 | `number` |
| `.min('列名')` | 求最小值 | `number` |
| `.exists()` | 判断是否存在 | `boolean` |
| `.value('SQL表达式')` | 自定义 SELECT 表达式（在查询上下文中执行任意 SQL 表达式） | `string \| number \| null` |
| `.toSQL()` | 生成 SQL（调试用） | `string` |

**多条件链式调用：**
```
{[db.重要人物表.where('状态', '存活').where('年龄', '>', 20).list('姓名')]}
```

**OR 条件：**
```
{[db.角色表.where('阵营', '友方').orWhere('阵营', '中立').list('姓名')]}
```

**IN 查询：**
```
{[db.背包物品表.whereIn('物品名称', ['铁剑', '钢盾', '药水']).count()]}
```

**BETWEEN 范围查询：**
```
{[db.角色属性表.whereBetween('攻击力', 30, 80).list('姓名')]}
```

**分组聚合：**
```
{[db.背包物品表.groupBy('物品类型').count()]}
```

**去重：**
```
{[db.角色表.distinct().list('阵营')]}
```

**NOT IN 查询：**
```
{[db.背包物品表.whereNotIn('物品名称', ['铁剑', '钢盾']).list('物品名称')]}
```

**模糊匹配：**
```
{[db.背包物品表.whereLike('物品名称', '%药水%').list('物品名称')]}
```

**分组聚合 + HAVING 过滤：**
```
{[db.背包物品表.groupBy('类别').having('COUNT(*) > 1').all()]}
```

**分页查询：**
```
{[db.角色表.orderBy('年龄', 'DESC').limit(10).offset(20).all()]}
```

**IS NULL / IS NOT NULL：**
```
{[db.背包物品表.whereNull('类别').count()]}
{[db.背包物品表.whereNotNull('类别').list('物品名称')]}
```

**自定义 SELECT 表达式（value）：**
```
{[db.背包物品表.where('类别', '武器').value('SUM(数量) * 2')]}
{[db.背包物品表.value('COUNT(*) * 100')]}
```

**中文表名/列名：** 自动通过 NameMapper 翻译为英文。

### 1.5 ORM 静态方法

#### `db.expr("SQL表达式")` — 执行任意 SQL 表达式

```
{[db.expr("3 + 5 * 2")]}
{[db.expr("(SELECT 数量 FROM 背包物品表 WHERE 物品名称='铁剑') * 2")]}
{[db.expr("(SELECT 数量 FROM 背包物品表 WHERE 物品名称='铁剑') + (SELECT 数量 FROM 背包物品表 WHERE 物品名称='钢盾') * 2") as total]}
```

- 支持任意 SQL 表达式：算术运算、子查询、聚合函数
- 支持中文表名/列名（自动翻译）
- 支持 `as 变量名` 语法，结果存入变量
- 无需先取值再计算，**一步完成跨表计算**

#### `db.rand(min, max)` — 生成随机整数

```
{[db.rand(1, 6)]}
{[db.rand(1, 100) as dice]}
```

- 生成 min 到 max 之间的随机整数（含两端）
- 使用 SQLite 的 `RANDOM()` 函数
- min > max 时自动交换
- 支持 `as 变量名` 语法

#### `db.calc("算术表达式")` — 执行含变量引用的算术表达式

```
{[db.calc("$v:sword_count + $v:shield_count * 2") as total]}
{[db.calc("($v:attack - $v:defense) * $v:dice") as damage]}
```

- 支持 `$v:变量名` 引用已定义的变量
- 通过 SQLite 执行计算，支持完整的 SQL 算术运算
- 变量不存在时返回空（不输出）
- 支持 `as 变量名` 语法

#### `db.max(值1, 值2, ...)` — 取多个值中的最大值

```
{[db.max(3, 7, 1)]}
{[db.max($v:a, $v:b, $v:c) as highest]}
```

- 支持任意数量的数值参数
- 支持 `$v:变量名` 引用（在执行前自动替换）
- 支持 `as 变量名` 语法
- 非数值参数被忽略

#### `db.min(值1, 值2, ...)` — 取多个值中的最小值

```
{[db.min(3, 7, 1)]}
{[db.min($v:a, $v:b, $v:c) as lowest]}
```

- 与 `db.max()` 相同，取最小值

---

### 2. 原生 SQL 查询 `{[sql "SQL语句"]}`

```
总数量: {[sql "SELECT SUM(quantity) FROM inventory"]}
```

- 支持中文表名/列名（自动翻译）
- 单值结果直接返回
- 多行单列用逗号分隔
- 多行多列用表格格式

---

### 3. ORM 查询存为变量 `{[db...as 变量名]}`

```
{[db.背包物品表.where('物品名称', '铁剑').get('数量') as sword_count]}
```

- 查询结果存入变量 `sword_count`，标签被移除（替换为空字符串）
- 后续通过 `$v:sword_count` 引用
- 数值结果自动转为数字类型，非数值保持字符串

---

### 4. SQL 查询存为变量 `{[sql "..." as 变量名]}`

```
{[sql "SELECT SUM(quantity) FROM inventory" as total_items]}
```

- 查询结果存入变量 `total_items`，标签被移除
- 多行多列结果只取第一行第一列的值
- 后续通过 `$v:total_items` 引用

## 三、条件 `<if>` 标签

### 基本语法

```html
<if 类型="表达式">满足条件时的内容</if>
<if 类型="表达式">满足条件时的内容<else>不满足时的内容</if>
```

**支持嵌套：**
```html
<if seed="战斗">
  <if cell="角色属性表/生命值/数值 > 50">生命充足</if>
</if>
```

**大小写不敏感：** `<IF SEED="...">` 等同于 `<if seed="...">`。

---

### 类型一览

| 类型 | 说明 | 表达式格式 |
|------|------|-----------|
| `seed` | 关键词匹配（匹配最新 AI 回复正文） | 关键词表达式 |
| `cell` | 表格单元格值比较 | 单元格引用 + 比较运算符 + 值 |
| `cond` | 统一条件表达式（支持所有前缀和逻辑运算） | 前缀条件表达式 |
| `cond` | 统一条件表达式（支持所有前缀和逻辑运算） | 前缀条件表达式 |
| `db` | ORM 数据库条件 | ORM 表达式 |
| `sql` | 原生 SQL 条件 | SQL 语句 |

---

### 3.1 `<if seed="关键词表达式">`

匹配最新一条 AI 回复的正文内容。

**关键词表达式语法：**

| 语法 | 说明 | 示例 |
|------|------|------|
| `关键词` | 简单匹配 | `<if seed="战斗">` |
| `关键词1,关键词2` | 或逻辑（任一匹配即为 true） | `<if seed="战斗,打架">` |
| `关键词1&关键词2` | 与逻辑（全部匹配才为 true） | `<if seed="战斗&主角">` |
| `!关键词` | 非逻辑（不包含该关键词为 true） | `<if seed="!魔法">` |
| `(表达式)` | 括号分组 | `<if seed="(战斗&主角),感情">` |

- 大小写不敏感
- 同时匹配 AI 回复正文和推进数据（plotContent）

---

### 3.2 `<if cell="单元格引用 运算符 值">`

比较表格单元格的值。

**单元格引用格式：**
- 三段式：`表名/行标识/列名`（精确匹配）
- 两段式：`表名/行名或列名`（模糊匹配，在行和列中搜索）

**支持的运算符：** `>`、`<`、`>=`、`<=`、`==`、`!=`

**全角运算符自动转换：** `＞` → `>`、`＜` → `<`、`＝` → `==`、`≥` → `>=`、`≤`/`≦` → `<=`、`≠` → `!=`

```html
<if cell="角色属性表/生命值/数值 > 50">生命充足</if>
<if cell="角色属性表/攻击力 >= 30">攻击力足够</if>
```

---

### 3.3 `<if db="ORM表达式">`

通过 ORM 查询构建器判断数据库条件。

```html
<!-- 判断是否存在 -->
<if db="db.背包物品表.where('物品名称', '铁剑').exists()">有铁剑</if>

<!-- 比较运算 -->
<if db="db.背包物品表.count() > 2">物品超过2种</if>

<!-- 省略 db. 前缀也可以 -->
<if db="背包物品表.count() > 0">有物品</if>
```

**truthy 判断规则（无比较运算符时）：**
- `null`、`undefined`、`''`、`'false'`、`'0'`、`0` → false
- 其他值 → true

---

### 3.4 `<if sql="SQL语句">`

通过原生 SQL 判断条件。

```html
<if sql="SELECT 1 FROM inventory WHERE item_name = '铁剑'">有铁剑</if>
<if sql="SELECT COUNT(*) FROM characters WHERE status = '存活'">有存活角色</if>
```

- 有结果且第一个值为 truthy → true
- 无结果或第一个值为 falsy → false
- 支持中文表名/列名（自动翻译）

---

### 3.5 `<if cond="统一条件表达式">`

最强大的条件类型，支持所有前缀和逻辑运算的组合。

**前缀一览：**

| 前缀 | 说明 | 示例 |
|------|------|------|
| `seed:` | 关键词匹配 | `seed:战斗` |
| `cell:` | 单元格值比较 | `cell:角色属性表/生命值/数值 > 50` |
| `random:` | 随机数变量比较 | `random:dice > 3` |
| `calc:` | 计算变量比较 | `calc:total > 10` |
| `max:` | 最大值变量比较 | `max:highest > 50` |
| `min:` | 最小值变量比较 | `min:lowest < 20` |
| `db:` | ORM 数据库条件 | `db:背包物品表.count() > 0` |
| `sql:` | 原生 SQL 条件 | `sql:SELECT COUNT(*) FROM inventory` |
| `v:` | 变量值比较 | `v:sword_count > 3` |
| （无前缀） | 默认作为关键词匹配 | `战斗`（等同于 `seed:战斗`） |

**逻辑运算符：**

| 运算符 | 说明 | 优先级 |
|--------|------|:------:|
| `()` | 括号分组 | 最高 |
| `!` | 取反（前缀） | 高 |
| `&` | 与（AND） | 中 |
| `,` | 或（OR） | 最低 |

**random: 的特殊语法 — 内联随机数：**
```html
<!-- 引用已定义的随机数变量 -->
<if cond="random:dice > 3">骰子大于3</if>

<!-- 内联生成随机数（不需要预先定义） -->
<if cond="random:1-100 > 50">50%概率触发</if>
```

**组合示例：**

```html
<!-- 与逻辑 -->
<if cond="seed:战斗 & cell:角色属性表/生命值/数值 > 50">战斗中且生命充足</if>

<!-- 或逻辑 -->
<if cond="seed:魔法, seed:战斗">使用了魔法或正在战斗</if>

<!-- 取反 -->
<if cond="!seed:死亡">角色未死亡</if>
<if cond="!db:背包物品表.where('物品名称', '铁剑').exists()">没有铁剑</if>

<!-- 括号分组 -->
<if cond="(seed:战斗 & cell:角色属性表/生命值/数值 > 50), seed:逃跑">
  战斗中且生命充足，或者正在逃跑
</if>

<!-- 混合 db: 和 seed: -->
<if cond="db:背包物品表.count() > 0 & seed:战斗">有物品且正在战斗</if>

<!-- 混合 db: 和 sql: -->
<if cond="db:背包物品表.where('物品名称', '铁剑').exists(), sql:SELECT 1 FROM characters WHERE status = '存活'">
  有铁剑或有存活角色
</if>

<!-- 复杂嵌套 -->
<if cond="(db:背包物品表.count() > 2 & !seed:死亡), random:1-100 > 80">
  （物品超过2种且未死亡）或 20%概率触发
</if>

<!-- 使用 v: 变量条件 -->
<if cond="v:sword_count > 0 & v:total > 10">有铁剑且总数超过10</if>
<if cond="v:dice > 3, v:chance > 50">骰子大于3 或 概率超过50%</if>
```

## 五、处理顺序

模板变量的处理按以下顺序执行：

1. **`<random>` 标签解析** — 生成随机数，存储变量
2. **`<calc>` 标签解析** — 计算表达式，存储变量
3. **`<max>` / `<min>` 标签解析** — 取最大/最小值，存储变量
4. **`$random:` / `$calc:` / `$max:` / `$min:` 变量引用替换** — 将变量引用替换为实际值
5. **`{[db...]}` / `{[sql...]}` / `$v:` 值替换** — 执行数据库查询、替换变量引用
   - 5a. `{[db...]}` / `{[db...as 变量名]}` — ORM 查询并替换/存储（含 db.expr/db.rand/db.calc/db.max/db.min 静态方法）
   - 5b. `{[sql "..."]}` / `{[sql "..." as 变量名]}` — SQL 查询并替换/存储
   - 5c. `$v:变量名` — 替换变量引用为实际值
6. **`<if>` 条件标签解析** — 根据条件选择输出内容

> ⚠️ 注意：由于处理顺序的关系，`<if>` 标签内的文本在条件判断时已经完成了变量替换。
> 
> ⚠️ 新变量系统（`$v:`）的处理顺序：`{[db...]}` → `{[sql...]}` → `$v:` 替换。
> `{[db...]}` 表达式内部支持 `$v:` 引用（在执行前自动替换），因此 db.calc/db.max/db.min 可以引用之前的 db 查询结果。
