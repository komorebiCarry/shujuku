# 目录结构说明

## 架构概述

本项目已完成三层架构重构，所有代码归入四个层级目录：

- **`src/shared/`** — 纯工具函数、常量、JSON 辅助（无副作用，零依赖）
- **`src/data/`** — 数据模型、存储后端、Repository（数据访问抽象）
- **`src/service/`** — 业务逻辑编排（AI 调用、世界书注入、表格更新、模板作用域等）
- **`src/presentation/`** — UI 组件、页面、窗口系统、启动引导

旧的 `src/core/`、`src/features/`、`src/ui/` 目录已全部删除。

---

## 构建方式

```bash
npx rollup -c
```

`rollup.config.js` 会：
1. 按依赖顺序编译 51 个 TS/JS 模块（TypeScript → JavaScript，去掉 export/import）
2. 在产物中包裹 IIFE + UserScript 头
3. 输出 `dist/index.bundle.js`

验证命令：
```bash
node --check dist/index.bundle.js        # 语法检查
node scripts/audit-bundle.cjs            # 793 标识符覆盖率 + UserScript 元数据
```

## 层级依赖规则

```
shared ← data ← service ← presentation
```

- `shared/` 不依赖任何其他层
- `data/` 可依赖 `shared/`
- `service/` 可依赖 `shared/` + `data/`
- `presentation/` 可依赖所有层

注意：当前产物仍为 IIFE 单文件，函数名在闭包内全局可见。层级依赖为逻辑约束，非编译时强制。
