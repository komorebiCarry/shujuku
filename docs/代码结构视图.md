## 四、各层职责说明

### shared 层（纯函数 / 常量 / 类型）
- **零副作用**：所有文件只包含纯函数、常量定义和类型声明
- **零依赖**：不 import 任何其他层（仅同层互引）
- `host-api.ts` 特殊：声明宿主API引用变量（`SillyTavern_API_ACU` 等），由 `state-manager.ts` 在运行时赋值

### data 层（数据存取 CRUD）
- **不含业务逻辑**：只做读/写/删/格式化
- **不持有业务状态**：需要 settings 的函数通过参数接收（如 `saveCurrentProfileTemplate_ACU(tpl, settings)`），不通过 DI 或回调间接获取
- `tavern-storage.ts`：统一存储入口（酒馆设置桥接 → IDB缓存 → localStorage 降级）
- `models/`：纯 TypeScript 接口，全项目共用的类型定义
- `gateways/`：外部API网关层，封装所有对宿主平台（TavernHelper/SillyTavern）的调用，service 层不再直接引用宿主 API
- `chat-message-data-repo.ts`：封装所有对 `message.TavernDB_ACU_*` 字段的底层 CRUD，service 层不再直接操作消息字段

### service 层（业务逻辑编排）
- **不操作 DOM**：所有 UI 操作由 presentation 层完成
- `helpers-remaining.ts`（136行入口）+ 5个子模块：辅助函数集，按职责拆分为上下文标签(context-tags)、表格锁定(table-lock)、数据合并(data-merge)、模板变量(template-vars)、剧情推进运行时(plot-runtime)
- `state-manager.ts`：全局运行时状态（settings、当前数据、API引用），通过 `_set_` 函数提供可控赋值

### presentation 层（UI渲染 / DOM操作 / 用户交互）
- **不直接访问 data 层**：全部通过 service 层间接操作
- `ui-refs.ts`：集中管理所有 jQuery 元素引用，通过 `_assignUIPlaceholders_ACU` 统一赋值
- `api-registry.ts`（~45行）+ `api-groups/`（9个文件）：暴露给酒馆前端的完整API注册表，按领域拆分
- `main-popup.ts`（1,413行）+ 8个标签页模块：主弹窗的 HTML 生成（按UI区域拆分）
- `popup-bindings.ts`（198行）+ 5个子模块：主弹窗事件绑定（按功能区域拆分）
