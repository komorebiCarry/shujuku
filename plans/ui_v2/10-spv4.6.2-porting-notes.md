# UI v2 移植到 spv4.6.2 的 API 适配记录

本文记录将 `test` 分支的 UI v2 移植到 upstream `main` / `spv4.6.2` 时，因为保留原项目 service、data、旧 UI 实现而产生的 API 适配点。

本次迁移策略：

- service、data、旧 UI 绑定层保留 upstream `spv4.6.2`。
- UI v2 不回退 service/data，不恢复 v4.0-beta 旧 API。
- UI v2 composable、store、page 后续按 `spv4.6.2` 当前 API 重新适配。
- `dist/index.bundle.js` 和根目录 `index.js` 由构建生成，不作为手工适配依据。

## 构建与入口

为支持 UI v2，构建层保留了 Vue 相关能力：

- `vue`
- `pinia`
- `unplugin-vue`
- `@rollup/plugin-replace`
- `src/presentation-v2/build/rollup-vue-script-transpiler.js`
- `src/presentation-v2/build/rollup-sfc-style-injector.js`

同时保留 `spv4.6.2` 的 extension 构建同步语义：

- extension 构建产物输出到 `dist/extension/index.js`。
- extension 构建完成后同步根目录 `index.js`。
- extension 构建完成后同步 `manifest.json` 到 `dist/extension/manifest.json`。

UI v2 后续适配时，不应绕开现有 `BUILD_MODE` 机制。

## 向量索引类型位置

`spv4.6.2` 使用共享模型作为向量索引类型来源：

```ts
src/shared/models/summary-vector-index.ts
```

UI v2 代码如果仍引用旧路径，需要统一调整：

```ts
// 旧路径，迁移时不要继续新增
src/service/vector/summary-vector-index-types.ts
src/shared/models/summary-vector-index-types.ts

// 当前主线路径
src/shared/models/summary-vector-index.ts
```

重点检查：

- `src/presentation-v2/composables/useVectorIndexConfig.ts`
- `src/presentation-v2/composables/visualizer/useVisualizerSave.ts`
- `tests/presentation-v2/vector-index/vector-index-page.test.ts`

## 向量索引缓存、删除、恢复 API

`spv4.6.2` 已把交火模式向量索引的缓存、删除、恢复和安全清理逻辑继续演进。UI v2 需要以当前 service 导出为准。

重点确认这些旧 API 是否仍被 UI v2 引用：

```ts
clearSummaryVectorIndexTempCache_ACU
deleteCurrentSummaryVectorIndexForCurrentChat_ACU
tryRecoverSummaryVectorIndexFromExternalSnapshot_ACU
patchCurrentVectorMemoryConfig_ACU
```

若存在引用，需要改为 `spv4.6.2` 当前语义对应的 API。当前主线中需要重点查看：

```ts
src/service/vector/summary-vector-index-cache-service.ts
src/service/vector/summary-vector-index-chat-service.ts
src/service/vector/vector-memory-config.ts
```

适配时需要确认：

- 清理缓存是只清临时缓存，还是同时清热缓存、flush task。
- 删除当前索引是否需要同步删除外置文件、tag data manifest、scope cache。
- 从外部快照恢复时是否需要按 chat、isolation、source table key 做多候选匹配。
- 向量记忆配置更新应使用当前主线的全局配置更新函数。

## 表格更新编排

`spv4.6.2` 的 `src/service/table/update-orchestrator.ts` 已包含新的表格更新流程，例如：

- AI 反馈合并前置。
- 分组对后续步骤透明。
- 当前表格持久化和 delta/checkpoint 逻辑。
- 表级 API preset override。

UI v2 后续需要复查这些调用点：

```ts
src/presentation-v2/composables/useManualUpdate.ts
src/presentation-v2/composables/useImportFlow.ts
src/presentation-v2/pages/ImportPage.vue
src/presentation-v2/pages/FormFillPage.vue
```

重点确认：

- `executeCardUpdateCore_ACU` 当前参数签名。
- `orchestrateManualUpdate_ACU` 当前参数签名。
- `requestOptions` 中 table preset override 的传递方式。
- import mode 是否仍不写入聊天记录。
- v2 “强制开启屏蔽外部导入世界书占位符”应继续留在 v2 import flow 内部，不能修改 service 层旧 UI 行为。

## 聊天 service 与表格持久化

`spv4.6.2` 的 `src/service/chat/chat-service.ts` 包含当前主线的聊天层业务逻辑，包括表格层清理、delta retention、向量索引外置文件清理等。

UI v2 需要避免依赖 v4.0-beta 旧行为，重点复查：

```ts
src/presentation-v2/composables/useDataManagement.ts
src/presentation-v2/composables/visualizer/useVisualizerSave.ts
src/presentation-v2/pages/DataMgmtPage.vue
```

需要确认：

- `saveIndependentTableToChatHistory_ACU` 当前签名。
- 清理表或清理楼层数据时是否会同步处理 table persistence v2。
- 可视化保存后是否仍需要触发向量索引 flush。

## 旧 UI 绑定层差异

以下旧 UI 文件保留 `spv4.6.2` 当前实现：

```ts
src/presentation/pages/popup-bindings-data.ts
src/presentation/pages/popup-bindings-worldbook.ts
```

UI v2 不应复制这些文件里的旧 API 调用，而应通过 composable/store 调用当前 service。

需要重点对照的功能：

- 交火模式开关。
- 交火索引状态刷新。
- 交火索引缓存清理。
- 当前索引删除。
- 旧索引迁移。
- 向量记忆配置保存。

## 测试适配

本次保留 `spv4.6.2` 的旧测试文件，不用 v4.0-beta 测试整文件覆盖。

后续如果需要恢复 v4.0-beta 中的测试修复，应只迁移具体补丁：

- `tests/presentation/visualizer-template-assistant.test.ts`
  - 可迁移 fake DOM 能力补丁，例如 `appendChild`、`remove`、`getElementById`。
  - 可按需要保留 `@vitest-environment jsdom`。

- `tests/service/table/update-orchestrator.test.ts`
  - 不整文件覆盖。
  - 只补当前主线仍适用的断言。
  - 重点覆盖旧 UI import toggle 与 v2 import 强制开启之间的行为边界。

## 建议验证

完成 UI v2 API 适配后，建议按顺序运行：

```bash
npm install
npm run typecheck
npm run build
node scripts/check-arch.mjs
npx vitest run tests/presentation-v2
npx vitest run tests/service/worldbook/injection-engine-state.test.ts
npx vitest run tests/service/ai/api-preset-selection.test.ts
```

最后再运行：

```bash
npm test
```

若全量测试失败，需要与 `spv4.6.2` 基线对比，区分主线既有失败和 UI v2 移植新增失败。
